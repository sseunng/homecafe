# 홈 카페 키오스크 (Home Cafe Kiosk) 개발 명세 및 지침

본 문서는 홈 서버(Proxmox) 환경에 배포하여 홈 카페에서 사용할 키오스크 웹 애플리케이션의 개발 요구사항, 기술 스택, UI/UX 디자인 방향 및 배포 전략을 정의합니다.

---

## 1. 프로젝트 개요 (Overview)
- **목적**: 홈 카페 방문객(게스트)이 QR 코드를 통해 접속하여 음료를 주문하고, 관리자(호스트)는 실시간으로 주문을 확인·제조하여 게스트에게 알림을 제공하는 실시간 웹 애플리케이션.
- **배포 환경**: 개인 홈 서버 내 Proxmox VM/LXC 컨테이너 (서버 IP: `192.168.1.38` 대역)
- **핵심 가치**: 
  - **편의성**: 가격 입력 없는 단순화된 주문 흐름.
  - **실시간성**: 웹소켓을 통한 주문 상태 변경의 실시간 알림.
  - **시각적 완성도**: 토스 오더(Toss Order) 스타일의 깔끔하고 트렌디한 UI/UX.

---

## 2. 기술 스택 제안 (Tech Stack)

### 프론트엔드 (Frontend)
- **Framework**: **Vite + React (or Vanilla JavaScript + CSS)**
  - 컴포넌트 기반 개발과 빠른 핫 리로딩을 위해 Vite + React가 유지보수 및 실시간 상태 관리에 유리합니다.
- **Styling**: **Vanilla CSS (Modern CSS)**
  - 토스 특유의 미니멀하고 고급스러운 UI 구현을 위해 커스텀 CSS 변수(HSL 색상계), Glassmorphism, CSS Grid/Flexbox, 미세 애니메이션(Micro-animations)을 적용합니다.
- **Real-time Sync**: **Socket.io-client** (실시간 주문 현황 송수신 및 웹 알림 처리)

### 백엔드 & 데이터베이스 (Backend & DB)
- **Server**: **Node.js (Express)**
  - 실시간 통신을 위한 Socket.io 서버 내장 및 API 서버 역할.
- **Database**: **SQLite 또는 파일 시스템 기반 JSON DB (Lowdb)**
  - 홈 카페 규모 특성상 무거운 DB 대신 가볍고 백업이 쉬운 SQLite나 JSON 파일 DB를 사용하여 메뉴 정보와 주문 이력을 관리합니다.
- **Image Storage**: 로컬 파일 시스템 업로드 처리 (Express static 폴더 활용)

## 3. 핵심 기능 정의 (Core Features)

### 3.1. 게스트 모드 (Guest Mode) - 기본 실행 모드
- **메뉴 브라우징**: 
  - 메뉴판 리스트 노출 (메뉴 이미지, 음료명, 설명(선택)).
  - 카테고리 탭 제공 (커피, 논커피, 티, 디저트 등).
- **장바구니 & 주문 흐름**:
  1. 메뉴 선택 시 하단에 **[장바구니 담기]** / **[바로 주문하기]** 슬라이드업 시트 또는 바텀 시트 노출.
  2. **[장바구니 담기]** 선택 시:
     - 토스 스타일의 토스트 팝업 표출: `"장바구니에 담았습니다!"`
     - 팝업 내 버튼: **[계속 담기]** (팝업 닫힘) / **[주문하기]** (주문 확인 창 이동)
  3. **장바구니 진입 경로**:
     - 상단 우측 고정 장바구니 아이콘 배지 클릭.
     - 메뉴 선택 후 바로 주문하기 / 담기 완료 팝업에서 주문하기 클릭.
  4. **주문 확인 & 완료**:
     - 담은 메뉴 목록 확인 및 최종 **[주문하기]** 버튼 클릭.
     - 주문 번호가 순차적으로 자동 부여됨 (예: `#101`, `#102`...).
     - 주문 완료 화면 유지 (현재 주문 상태 표시: "주문 대기 중").
- **알림 수신**:
  - 브라우저 알림 권한 허용 시 또는 웹 화면 내에서 실시간 알림 팝업 작동.
  - 관리자가 완료 처리 시: `"☕ {주문번호}번 음료가 완성되었습니다! 픽업대로 와주세요."` 안내 및 소리 알림.

### 3.2. 관리자 모드 (Admin Mode)
- **주문 관리 (실시간 주문 현황)**:
  - 실시간으로 들어오는 주문 목록을 카드 형태로 시각화.
  - 주문 상태 변경 흐름:
    1. **주문 대기 (Pending)**: 게스트가 주문을 완료한 직후 상태. **[수락]** 버튼 제공.
    2. **준비 중 (Preparing)**: 호스트가 수락을 누른 상태 (기존 '제조 중'보다 직관적이고 따뜻한 톤앤매너로 **"맛있게 만드는 중 ☕"** 권장). **[완료]** 버튼 제공.
    3. **제조 완료 (Completed)**: 호스트가 완료를 누른 상태. 해당 게스트 웹 브라우저로 실시간 알림 송신 및 이력으로 이동.
- **주문 이력 (History)**:
  - 완료되거나 취소된 과거 주문 리스트 조회.
  - 오늘의 일일 주문 통계 (총 주문 건수, 인기 메뉴 등 간단한 통계).
- **메뉴 관리 (Menu Management)**:
  - **신규 메뉴 추가**: 음료명, 설명(옵셔널), 메뉴 이미지 업로드 기능.
  - **기존 메뉴 편집/삭제**: 품절 처리(Sold out 활성화/비활성화) 및 영구 삭제 기능.

## 4. UI/UX 디자인 방향 (Toss Order Style)
토스 특유의 '쉽고, 직관적이며, 시각적으로 편안한' 디자인 시스템을 차용합니다.

- **Color Palette (HSL 기반)**:
  - Primary: Toss Blue (`#0064FF` 또는 HSL을 활용한 생동감 넘치는 블루)
  - Background: 깨끗한 White 바탕과 Soft Gray (`#F2F4F6`), 다크 모드 지원 고려.
  - Text: 완전한 Black 대신 Deep Charcoal (`#191F28`) 사용으로 가독성 극대화.
- **Typography**:
  - `Toss Product Sans` 또는 `Inter` / `Pretendard` 서체 활용.
  - 굵고 명확한 타이포그래피 계층 구조 설계.
- **Visual Elements**:
  - 모서리가 둥근 둥글둥글한 카드 UI (Border Radius: `16px` ~ `24px`).
  - 버튼 클릭 시 약간 작아졌다가 커지는 미세한 스케일 애니메이션 (`active: scale(0.96)`).
  - 바텀 시트(Bottom Sheet)가 아래에서 부드럽게 올라오는 인터랙션.
  - 불필요한 테두리를 줄이고 여백(Padding)과 옅은 그림자(Box Shadow)를 활용한 공간 분할.

## 5. 관리자/게스트 접속 분리 및 보안 아이디어

홈 카페 내부 망 또는 외부 공유 시 게스트가 관리자 화면을 오작동시키는 것을 방지해야 합니다. 다음 두 가지 방안을 병행하여 적용합니다.

### 1안: 숨겨진 제스처 + PIN 번호 인증 (권장)
- **설명**: 게스트 모드 화면에서 특정 영역(예: 좌측 상단 로고)을 **3초간 길게 누르거나(Long Press) 5회 연속 탭**하면, 관리자 진입을 위한 PIN 패드가 나타납니다.
- **보안**: 설정해 둔 4자리 PIN 번호(예: `2026`)를 입력해야만 관리자 세션이 활성화되고 `/admin` 페이지로 리다이렉트됩니다.
- **장점**: 게스트 화면에 관리자 버튼이 직접 노출되지 않아 UI가 극도로 깔끔해지고, 아이들이 호기심에 접속하는 것을 방지할 수 있습니다.

### 2안: 별도 경로 및 IP 대역/비밀번호 제한
- **설명**: 관리자 페이지의 URL을 `/admin` 혹은 유추하기 어려운 `/host-portal`로 지정합니다.
- **보안**: 
  - 최초 접속 시 간단한 패스워드 인증을 요구하며, 성공 시 브라우저 쿠키/LocalStorage에 토큰을 저장합니다.
  - 로컬 네트워크 환경(192.168.x.x)에서만 관리자 로그인을 허용하도록 백엔드 middleware 수준에서 보호 조치를 적용합니다.

## 6. 홈 서버(Proxmox) 배포 전략

1. **Docker 컨테이너화**:
   - 프론트엔드와 백엔드를 단일 Docker 이미지 또는 `docker-compose`로 구성합니다.
   - 데이터와 업로드 이미지는 Proxmox 호스트 디렉토리와 마운트(Volume)하여 영구 보존합니다.
2. **Proxmox LXC 컨테이너 배포**:
   - 가볍고 자원을 적게 먹는 Ubuntu/Alpine 기반 LXC 컨테이너를 생성합니다.
   - 컨테이너 내부에 Node.js와 Docker를 구성하여 서비스를 구동합니다.
3. **Nginx Reverse Proxy & SSL 구성 (옵션)**:
   - Proxmox 환경 내 Nginx Proxy Manager나 Caddy를 활용해 SSL 인증서(HTTPS)를 적용합니다.
   - 로컬 도메인(예: `kiosk.local` 또는 `homecafe.my`)을 부여하여 접속을 용이하게 합니다.
4. **QR 코드 생성**:
   - 게스트가 스캔하면 모바일 브라우저로 키오스크 페이지(`https://192.168.1.XX` 또는 로컬 도메인)에 즉시 접속할 수 있는 QR 코드를 제작하여 카페 테이블이나 벽면에 배치합니다.
